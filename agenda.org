* Packaging

** DONE Split bin/ scripts into src/hypervariorum package + thin wrappers
:PROPERTIES:
:CUSTOM_ID: packaging/src-layout-split
:END:
Settled per
[[file:doc/forum.org::#architecture/ingestion-model-classifier-split][#architecture/ingestion-model-classifier-split]]: =src/hypervariorum/=
splits into =nvs/= (NVS-specific: markup parsing, entity conversion),
=model/= (source-agnostic: Annotation, the dependency graph), and
=classifiers/= (source-agnostic: the AI-assisted edge classifiers).
=nvs/= stays singular for now — no =sources/= family until a second
source corpus actually appears.

Create that layout under =src/hypervariorum/=. Move the pure
logic out of =bin/parse_annotations.py= and =bin/convert_entities.py=
into =src/hypervariorum/annotations.py= and
=src/hypervariorum/entities.py= respectively; leave =bin/= holding
only the stdin/stdout wiring (the =if __name__ == "__main__":=
block), so the scripts stay usable as Makefile filter targets.

Also needed:
- Replace leftover =my_python_project= placeholders in
  =pyproject.toml= (=[tool.pdm.scripts]=, =[tool.pytest.ini_options]=)
  with =hypervariorum=.
- Add the new =src/hypervariorum/*.py= files as prerequisites on the
  relevant Makefile rules, so edits to the real logic trigger a
  rebuild, not just edits to the =bin/= wrapper.
- Add unit tests in =tests/= for =roman_to_int=,
  =parse_annotations_to_xml=, =convert_entities=, and =process_line=
  now that they're importable.

Completion criterion: =pdm run pytest= passes with real tests
against the moved functions, and =make= still produces identical
output from =raw/= as before the split.
